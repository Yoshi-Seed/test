# 医療従事者向けパネル 簡易調査画面ビルダー

Phase1.5 プロトタイプ - フロントエンドのみで動作する調査ビルダー

## 🎯 概要

医療従事者向けパネル調査を簡単に作成できるビルダーツールです。
- **バックエンド不要**: localStorageで完結
- **1問1画面方式**: 回答者フローを実際に体験できる
- **条件分岐対応**: スキップ・終了などの分岐ルールをGUIで設定
- **JSON Import/Export**: 設定をJSONで保存・共有

## ✨ 主な機能

### 📝 対応している質問タイプ (5種類)

| タイプ | アイコン | 説明 |
|--------|---------|------|
| **SA** | ⭕ | 単一選択（ラジオボタン） |
| **MA** | ☑️ | 複数選択（チェックボックス） |
| **FREE** | 📝 | 自由記述（テキストエリア） |
| **NUMERIC** | 🔢 | 数値入力（整数のみ） |
| **MATRIX** | 📊 | マトリクス（行×列選択） |

### 🔀 条件分岐ルール

- **NEXT**: 次の質問へ（デフォルト）
- **GOTO**: 指定QIDへスキップ
- **SCREENOUT**: 対象外終了
- **COMPLETE**: 完了
- **QUOTA_FULL**: 割付満了

### ⚠️ バリデーション・警告

- 参照先QIDが存在しない → 警告表示
- ループの可能性検出 → 警告表示
- 必須項目の未入力 → 未完成マーク
- Matrix行列の空チェック

## 🚀 セットアップ

### 1. インストール

```bash
cd survey-builder
```

### 2. 開発サーバー起動

```bash
npm run dev
```

ブラウザで http://localhost:3001 にアクセス

## 📖 使い方

### 基本フロー

1. **質問追加**: 左パネルの「➕ 質問を追加」ボタンをクリック
2. **タイプ選択**: 5つの質問タイプからカードを選択
3. **質問編集**: 右パネルで質問文・選択肢などを入力
4. **プレビュー**: 「👁️ プレビュー」ボタンで回答者視点を確認
5. **Export**: 完成したらJSONをエクスポート

### 画面構成

#### ビルダー画面 (`/builder`)

**2ペイン構成**:
- **左ペイン**: 質問一覧
  - QID + 状態アイコン（✅完成 / ⚠️未完成 / 🔀分岐あり）
  - 上下移動・削除ボタン
  - 質問追加ボタン

- **右ペイン**: 質問編集フォーム
  - QID編集可能
  - 質問文（必須）
  - 補足文（任意）
  - 必須フラグ
  - タイプ別の入力UI
  - 条件分岐ルール設定
  - ミニプレビュー（常時表示）

#### プレビュー画面 (`/preview`)

- 1問1画面方式
- 戻るボタン無し
- バリデーション動作
- 分岐ルールが効く
- 完了/対象外/割付満了の終了画面

### 選択肢の設定（SA/MA）

1. 選択肢セクションで「➕ 選択肢を追加」
2. **value**: 内部値（1,2,3...）
3. **label**: 表示テキスト
4. **その他チェック**: ONにすると回答時にテキスト入力欄が表示

### 条件分岐ルールの設定

1. 「条件分岐ルール」セクションで「➕ ルールを追加」
2. 自然文表示: `IF S3 = "1" → THEN 対象外終了`
3. 「編集」ボタンでアクションを変更
4. GOTO選択時は遷移先QIDを入力

### Import / Export

#### エクスポート
```
「📥 エクスポート」ボタン → JSONファイルがダウンロード
```

#### インポート
```
「📤 インポート」ボタン → JSONを貼り付け → 「インポート」
```

## 🎓 サンプルデータ

初回起動時、または「📄 サンプル読込」ボタンで以下のサンプルが読み込まれます:

### サンプル調査の内容
- **S1**: 職種（SA）
- **S2**: 施設形態（SA）
- **S3**: 経験年数（SA）**← 分岐ルールあり**
  - 1年未満 → SCREENOUT
  - 5年以上10年未満 → GOTO S5（S4をスキップ）
- **S4**: 医療機器（MA + その他オプション）
- **S5**: 改善必要なこと（FREE）
- **S6**: 勤務時間（NUMERIC）
- **S7**: 職場環境評価（MATRIX）

```json
        { "value": "1", "label": "選択肢1" },
        { "value": "2", "label": "選択肢2" }

          "when": { "op": "equals", "value": "1" },
          "then": { "action": "SCREENOUT" }

    "completeMessage": "完了メッセージ",
    "screenoutMessage": "対象外メッセージ",
    "quotaFullMessage": "割付満了メッセージ"
```

## 🎬 デモフロー

### 5分で試せる基本フロー

1. **起動**: `npm run dev` → http://localhost:3001
2. **サンプル読込**: 「📄 サンプル読込」ボタン
3. **質問確認**: 左のリストからS3を選択 → 分岐ルールを確認
4. **プレビュー**: 「👁️ プレビュー」ボタン
5. **分岐体験**: S3で「1年未満」を選択 → 対象外画面へ遷移
6. **再テスト**: 「最初からやり直す」で別の選択肢を試す

### 質問作成フロー

1. **追加**: 「➕ 質問を追加」
2. **タイプ**: MAを選択
3. **質問文**: 「好きな色を全て選んでください」
4. **選択肢**: 赤・青・黄・その他
5. **その他**: 「その他」にチェック
6. **プレビュー**: 動作確認

## 🐛 トラブルシューティング

### 質問が保存されない
→ ブラウザのlocalStorageが無効になっている可能性があります

### プレビューで「質問が見つかりません」
→ ビルダーに戻ってデータを確認してください

### JSONインポートエラー
→ JSONフォーマットが正しいか確認してください

### 警告が消えない
→ 指摘されている箇所（QID/選択肢/分岐先）を修正してください
